# 접근성 (A11y) 가이드

이 문서는 실시간 Q&A 애플리케이션의 접근성 기능과 개선 사항을 설명합니다.

## 구현된 접근성 기능

### 1. ARIA 레이블 및 역할

- **시맨틱 HTML**: 모든 주요 섹션에 적절한 HTML 요소와 ARIA 역할 사용
- **ARIA 레이블**: 모든 인터랙티브 요소에 명확한 `aria-label` 제공
- **ARIA 상태**: `aria-pressed`, `aria-required`, `aria-invalid` 등 상태 표시
- **ARIA 라이브 영역**: 동적 콘텐츠 업데이트를 위한 `aria-live` 속성

#### 주요 컴포넌트별 ARIA 구현:

- **QuestionCard**: `role="article"`, `aria-labelledby` 사용
- **투표 버튼**: `aria-pressed`, `aria-label`로 상태 및 기능 설명
- **폼 입력**: `aria-required`, `aria-label`로 필수 필드 및 용도 명시
- **에러 메시지**: `role="alert"`, `aria-live="assertive"` 사용
- **성공 메시지**: `role="status"`, `aria-live="polite"` 사용

### 2. 키보드 네비게이션

- **포커스 표시**: 모든 인터랙티브 요소에 명확한 포커스 링 표시
- **Tab 순서**: 논리적인 탭 순서로 요소 배치
- **키보드 단축키**: Enter/Space 키로 버튼 활성화 지원
- **스킵 링크**: 메인 콘텐츠로 바로 이동할 수 있는 스킵 링크 제공

### 3. 포커스 관리

- **포커스 트랩**: 다이얼로그 열릴 때 포커스가 다이얼로그 내부로 이동
- **포커스 복원**: 다이얼로그 닫힐 때 이전 포커스 위치로 복원
- **포커스 표시**: `focus-visible`을 사용한 명확한 포커스 링

### 4. 스크린 리더 지원

- **시맨틱 마크업**: 적절한 HTML 요소 사용 (`<time>`, `<article>` 등)
- **aria-hidden**: 장식용 아이콘에 `aria-hidden="true"` 적용
- **aria-live**: 동적 콘텐츠 업데이트를 스크린 리더에 알림
- **명확한 레이블**: 모든 입력 필드와 버튼에 명확한 레이블 제공

### 5. 색상 대비

- **WCAG AA 준수**: 텍스트와 배경 간 최소 4.5:1 대비율 유지
- **포커스 표시**: 색상뿐만 아니라 윤곽선으로도 포커스 표시
- **에러 표시**: 색상뿐만 아니라 아이콘과 텍스트로도 에러 표시

### 6. 터치 타겟 크기

- **최소 크기**: 모든 인터랙티브 요소 최소 44x44px (모바일)
- **터치 최적화**: `touch-action: manipulation` 적용

## 접근성 테스트

### 키보드 테스트

1. **Tab 키**: 모든 인터랙티브 요소를 Tab으로 순회 가능한지 확인
2. **Enter/Space**: 버튼과 링크가 키보드로 활성화되는지 확인
3. **Escape**: 다이얼로그가 Escape 키로 닫히는지 확인

### 스크린 리더 테스트

권장 도구:
- **macOS**: VoiceOver
- **Windows**: NVDA 또는 JAWS
- **Chrome**: ChromeVox 확장 프로그램

테스트 시나리오:
1. 페이지 로드 시 주요 콘텐츠가 읽히는지 확인
2. 폼 입력 시 레이블과 에러 메시지가 읽히는지 확인
3. 버튼 클릭 시 상태 변경이 알려지는지 확인

### 자동화 테스트

```bash
# Playwright 접근성 테스트 (추가 예정)
npm run test:e2e:accessibility
```
